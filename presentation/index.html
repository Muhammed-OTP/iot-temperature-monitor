Introduction

Problématique & motivation

La température est un paramètre critique dans l'industrie, la santé et la vie quotidienne. Sa surveillance automatisée est un besoin fondamental.

Sécurité industrielle

Les machines industrielles, serveurs informatiques et entrepôts pharmaceutiques nécessitent une surveillance continue de la température pour prévenir les pannes, les pertes de stock et les risques d'incendie.

Santé et agriculture

Chaîne du froid pour les médicaments et vaccins, surveillance des serres agricoles, contrôle du confort thermique dans les bâtiments — la température impacte de nombreux domaines critiques.

Solution IoT à faible coût

Un Arduino Uno (~5€) + un capteur TMP36 (~1€) = un système de surveillance fonctionnel pour moins de 10€. Accessible, reproductible et extensible.

Fondamentaux

L'Internet des Objets (IoT)

L'IoT connecte le monde physique au numérique — des capteurs qui sentent, un cerveau qui traite, des actuateurs qui réagissent.

Architecture 4 couches

1. Perception — Capteurs (température, humidité, mouvement)
2. Réseau — WiFi, Bluetooth, LoRa, Zigbee
3. Traitement — Microcontrôleur / Serveur
4. Application — Interface utilisateur / Dashboard

Analogie : le corps humain

Capteurs = organes sensoriels (peau, yeux, oreilles)
Microcontrôleur = cerveau qui analyse
Actuateurs = muscles qui réagissent
Communication = système nerveux

Notre projet implémente les couches perception (TMP36) et traitement (Arduino). Pour devenir IoT complet, il faudrait ajouter un module WiFi (ESP8266/ESP32).

Composants

Composants du système

Six composants essentiels pour un prototype fonctionnel — chacun a un rôle précis dans la chaîne Entrée → Traitement → Sortie.

Capteur TMP36

Capteur analogique de température. Plage : -40°C à +125°C. Précision : ±1°C. Sortie : 10mV/°C avec offset 500mV. Formule : T = (V-0.5)×100.

LED Verte

Indicateur visuel état normal. S'allume quand la température est sous le seuil. Connectée via résistance 220Ω pour limiter le courant à ~13.6 mA.

LED Rouge

Indicateur visuel état d'alerte. S'allume quand la température dépasse le seuil. Même protection par résistance 220Ω.

Buzzer Piézoélectrique

Alarme sonore. Activé par tone(pin, 1000Hz). Cristal piézo qui vibre à la fréquence du signal carré généré par l'Arduino.

Résistances 220Ω (×2)

Protection des LEDs. Loi d'Ohm : I = (5V-2V)/220Ω ≈ 13.6 mA. Sans résistance, le courant détruirait la LED (max 20 mA).

Breadboard + Fils

Plaque d'essai sans soudure pour le prototypage rapide. Rails d'alimentation (+5V, GND) et groupes de connexion horizontaux.

TMP36 — Brochage

Vue de face (partie plate vers vous) :
Gauche = VCC (5V) · Centre = Signal (A0) · Droite = GND
⚠️ Si inversé → valeurs aberrantes (-50°C) et risque de dommage.

Code Arduino

Implémentation logicielle

Structure standard Arduino : setup() pour l'initialisation, loop() pour la boucle infinie de lecture et de décision.

// Constantes
const int CAPTEUR = A0;
const int LED_VERTE = 3;
const int LED_ROUGE = 4;
const int BUZZER = 5;
const float SEUIL = 30.0;

void setup() {
  Serial.begin(9600); // Com. série
  pinMode(LED_VERTE, OUTPUT);
  pinMode(LED_ROUGE, OUTPUT);
  pinMode(BUZZER, OUTPUT);
}

void loop() {
  // 1. Lire valeur brute (0-1023)
  int brute = analogRead(CAPTEUR);

  // 2. Convertir en tension
  float tension = brute * (5.0/1024.0);

  // 3. Convertir en °C
  float temp = (tension - 0.5) * 100.0;

  // 4. Logique de décision
  if (temp < SEUIL) {
    digitalWrite(LED_VERTE, HIGH);
    digitalWrite(LED_ROUGE, LOW);
    noTone(BUZZER);
  } else {
    digitalWrite(LED_VERTE, LOW);
    digitalWrite(LED_ROUGE, HIGH);
    tone(BUZZER, 1000);
  }
  delay(1000); // 1 seconde
}

Discussion

Défis & limitations

Tout prototype a des contraintes — les documenter démontre la maturité technique et oriente les améliorations futures.

Pas de connectivité

Le système ne transmet pas les données via WiFi ou Bluetooth. Il n'est pas « connecté » au sens IoT complet. Solution : ajouter un ESP8266/ESP32.

Seuil fixe (30°C)

Le seuil est codé en dur — il faut reprogrammer pour le modifier. Solution : potentiomètre ou boutons pour ajustement dynamique.

Pas de stockage

Aucun historique des mesures n'est conservé. Les données disparaissent à l'arrêt. Solution : carte SD ou envoi vers un cloud.

Précision ±1°C

Le TMP36 a une précision limitée à ±1°C, insuffisante pour le médical. Solution : utiliser un DHT22 (±0.5°C) ou un thermocouple.

Pas d'affichage local

Les données ne sont visibles que via le moniteur série (PC nécessaire). Solution : écran LCD 16×2 ou OLED.

Oscillation au seuil

Si la température fluctue autour de 30°C, le système alterne rapidement. Solution : implémenter une hystérésis (ex: alerte à 30°C, retour à 28°C).

Perspectives

Améliorations futures

Plusieurs axes pour transformer ce prototype en un système IoT complet et déployable.

Connectivité WiFi

Remplacer Arduino par un ESP32 (WiFi intégré). Envoyer les données vers ThingSpeak ou Firebase. Dashboard web temps réel pour visualiser les courbes de température à distance.

Écran LCD / OLED

Affichage local de la température sans PC. Écran LCD 16×2 via I2C (2 broches seulement : SDA sur A4, SCL sur A5). Affichage autonome même sans ordinateur.

Notifications SMS/Email

Alertes envoyées par IFTTT, Telegram Bot ou Twilio SMS quand le seuil est dépassé. Prévenir même à distance, 24h/24.

Stockage de données

Module carte SD pour enregistrer l'historique avec horodatage. Export CSV pour analyse dans Excel ou Python (Pandas, Matplotlib).

Multi-capteurs

Ajouter humidité (DHT22), pression (BMP280), qualité d'air (MQ-135). Transformer le système en station météo complète.

Intelligence Artificielle

Prédiction de tendances par séries temporelles. Détection d'anomalies par ML. Seuil adaptatif auto-appris. TinyML directement sur microcontrôleur.
